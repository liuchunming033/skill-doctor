## 核心观点

AI Agent 正站在一个转折点上——从"会聊天"转向"能做事"。Anthropic 于今年10月16日发布了官方文档《Equipping agents for the real world with Agent Skills》，提出了 Skills 机制来解决 Agent 在真实世界任务中的瓶颈。

## Agent 的两大能力缺口

原文首先肯定了 Claude 的强大，但也指出在现实工作中它缺少两样关键能力：

1. **缺乏过程知识（Process Knowledge）**——不知道事情该怎么做。就像新员工会问"我们的报销流程是什么？""我该如何提交代码？"
2. **缺乏组织背景（Organizational Context）**——不知道东西在哪里。就像新员工会问"我们的项目API密钥在哪里？""TTT模板在哪？"

可以把一个 Agent 想象成一个聪明但没经验的新员工——虽然聪明，但入职第一天什么都做不了。

## Skill 的核心定义

Skill 是由**指令（Instructions）、脚本（Scripts）和资源（Resources）** 组成的结构化文件夹，智能体能够动态发现并加载这些内容，以提升特定任务的表现。

- **Instructions**：相当于手册中的 SOP 文字，告诉 Agent 如何一步步完成任务
- **Scripts**：需要执行的 Python 文件
- **Resources**：执行过程中用到的配置文件或模板（如 JSON 或 docx 文件）

这三个要素正好对应两个痛点：过程知识的缺失由指令和脚本解决，组织背景的缺失由资源解决。

## 关键特性：动态加载

Skill 不再被固定在系统提示中，而是可以被**动态检索、挂载和使用**。当 Claude 面对不同任务时（如分析财报、提取合同字段、生成合规文档），它会主动发现最相关的 Skill，加载其中的 SOP、脚本和模板，使输出更精准、更一致、更可复现。

Skill 不只是知识的存储单元，而是让 Agent 在特定场景下变得更专业的能力模块。它赋予了 Claude 在需要时加载能力模块的"心智模式"，让智能边界不再受上下文窗口限制。

## 系统架构

架构分为两侧：

1. **右侧——Agent 虚拟机（执行空间）**：Agent 的"电脑和双手"，是真正执行任务的地方。包含 Bash、Python 等执行引擎，Agent 可以通过它操作文件、运行脚本。

2. **左侧——Agent 配置（认知空间）**：LLM 是大脑、决策中心。配置中包含核心系统提示以及一份已装备技能的清单（技能名称和简介）。当 Claude 想使用某个 Skill（如 PDF）时，它会通过 Tool 操作虚拟机执行命令——LLM 指挥虚拟机打开并读取 SOP 文件，读取到的内容返回给 LLM 的上下文窗口，Claude 便能学习并决定下一步该怎么做。

## Skill vs Tool 的核心区别

| 维度 | Tool | Skill |
|------|------|-------|
| 回答的问题 | What（我能做什么） | How（我该如何做好这件事） |
| 技术角色 | 执行代码 | 指导如何完成任务 |
| 粒度 | 原子化能力（一个API或函数） | 完整的SOP（包含指令、脚本和资源） |
| 行为模式 | 执行器，被动等待调用 | 流程编排者，主动指导LLM调用Tool完成任务 |
| 目标 | 提供原子能力 | 封装可移植、可复用的过程知识 |

**类比**：Tool 像一台烤箱（底层执行能力），Skill 像一份妈妈的烤鸡食谱（封装了全部过程知识——烤多久、温度多少、需要哪些原料，甚至经验性的诀窍）。

Skill 不是 Tool 的替代品，而是调用 Tool 的 SOP。它封装了可组合的过程知识，让 Agent 真正具备"会做事"的能力。
