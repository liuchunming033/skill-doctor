## 核心观点

前两期讲清楚了 Skill 的原理（让智能体从会聊天变成能做事）和渐进式披露机制（实现几乎无限的上下文）。本期来到 Anthropic 文档的最后也是最重要的部分——如何真正编写、评估并安全地使用自己的 Skill。

## 一、从评估开始：用问题驱动建设

在构建 Skill 之前，要让智能体自己暴露问题——让它去执行真实任务（如填写表单、生成报告、分析合同），仔细观察它在哪些地方出错、卡顿或频繁需要额外提示。**这些错误和瓶颈就是 Skill 设计的起点。**

Anthropic 建议：
- **用问题驱动建设**，不要空想功能，而是用任务中的失败样本去反推缺陷
- **每一个 Skill 都应是针对具体能力短板的修复**
- **小步走**：先写出一个简单的版本验证可行性，再通过多轮使用逐步扩展内容。Skill 的成长来自持续验证，而不是一次写完。

## 二、为规模而设计：分层文件结构

当 Skill 变得复杂时，一个文件往往不够用了。要学会拆分：

- **最上层（元数据）**：`SKILL.md` 开头的名称和简介，约 100 个 token
- **中间层（正文）**：主要步骤和说明，通常在 5000 个 token 以内
- **底层（附属文件）**：脚本、参考文档等，由 Claude 按需读取

通过这种分层方式，智能体在使用时只加载需要的部分，既节省算力，又能随时扩展。

**特别注意**：Skill 中的代码既可以被执行，也可以被阅读。关键是让 Claude 明白哪些脚本要运行、哪些只是参考说明。这延续了前一集的核心思想——让思考和执行分离。

## 三、从 Claude 的角度思考

Skill 的作者是人类，但使用者是 Claude。在设计 Skill 时，要考虑它是如何理解和选择技能的：

- Claude 判断是否调用某个 Skill 时，**主要依赖 name 和 description 这两个字段**
- 它会基于这些描述决定哪个技能与当前任务最匹配
- **Skill 的命名和简介不是装饰，而是 Claude 做出决策的信号**
- 一句准确的描述，往往决定了一个技能能否被正确使���

同时应观察 Claude 在实际使用中的行为——是否误触发了技能？是否在某个场景过度依赖特定的上下文？这些观察结果都是改进 Skill 的依据。

## 四、与 Claude 一起迭代：人机共创

Anthropic 提出一个非常重要的理念：**Skill 的成长应该是人机共同完成的过程。**

- 当 Claude 成功完成任务时，让它自己总结哪些做法有效、哪些提示最关键，然后把这些总结写回 Skill 中，让它成为下一次执行的标准步骤
- 当它出错时，让它反思哪里出错、为什么误解、缺少了什么信息
- 通过这种反思式迭代，Claude 会逐渐学会如何改进自己的行为，Skill 也会随着经验不断进化

这就是一种**人机共创式学习**，让经验真正沉淀为 SOP。

## 五、安全使用

Skill 的强大同时也意味着风险——因为它可以运行脚本、访问文件系统，甚至连接外部网络。Anthropic 提供了三条安全建议：

1. **只安装可信来源的技能**
2. **使用前要仔细阅读技能中的所有文件**，特别是脚本依赖和外部资源链接
3. **理解它的执行逻辑**，明确哪些代码需要运行，哪些只是文档说明

Skill 的开放性让 Claude 真的能做事，但同时也需要我们具备安全工程师的意识。**安全不是限制能力，而是保护信任。**

## 六、未来展望：Skill 与 MCP

### Skill 共享与发现生态
未来个人和组织都能创建、编辑、发现、共享并使用 Skill，形成一个完整的生命周期。Claude 将不再只是一个人的助手，而是能访问全社会技能库的智能体。个人可以上传自己的专业技能模块，企业可以共享内部 SOP、API 和工作流。这将让知识从孤立变为互联，让不同的智能体之间形成真正的知识网络。

### Skill 与 MCP 的关系
两者并不是重复，而是互补：
- **Skill** 提供任务知识，教 Claude 如何完成一类工作（思维的框架）
- **MCP** 提供外部连接，让 Claude 可以访问数据库、API 和软件资源（行动的手臂）

当两者结合时，Claude 就能实现混合式智能——既懂得做事的方法，又能真正动手去执行。例如一个合同审查 Skill 可以通过 MCP 连接企业数据库，提取合同条目，自动比对风险字段，最后输出完整的审查报告。

### 更远的愿景
未来 Agent 将能自主创建、编辑并评估自己的技能，把反复验证的思维与操作模式转化为可复用的能力模块。当那一天到来时，**Skill 将不仅是人类写给智能体的说明书，更是智能体写给自己的成长记录。**
