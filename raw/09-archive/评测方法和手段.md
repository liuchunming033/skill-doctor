# 评测方法和手段

在智能体效果工程中，评测方法决定了指标如何落地执行。不同评测方法在**成本、稳定性、规模化能力、可信度**方面存在差异。通常需要组合使用多种评测手段，以保证评估结果的全面性和可靠性。

常见评测方法包括：

- 自动化评测（Automated Evaluation） 

- 人工评测（Human Evaluation） 

- 裁判模型评测（LLM\-as\-a\-Judge）

# 自动化评测（Automated Evaluation）

自动化评测是AI效果工程的基础能力，通过程序或算法自动计算评测指标，具有**低成本、高效率、可持续运行**等优点。

自动化评测通常基于标准测试集（benchmark dataset），对模型输出进行批量评估。常见方法：

## **Rule\-based 评测**

基于规则或数学公式进行评测，适用于有标准答案或可计算目标的任务。典型指标包括：

- **nDCG（Normalized Discounted Cumulative Gain）**
衡量排序结果质量，常用于RAG检索评测。

- **MRR（Mean Reciprocal Rank）**
衡量正确答案在检索结果中的排名情况。

- Accuracy / Precision / Recall / F1\-score
常用于分类、抽取、匹配类任务。

**优点：**

- 稳定性高

- 可重复性强

- 便于横向比较

**局限：**

- 难以评估开放式生成任务

- 对复杂语义理解能力有限

## **基于语义相似度的评测**

通过Embedding或语义匹配算法计算生成结果与参考答案的相似度。

示例：

- BERTScore

- Cosine similarity

- Semantic textual similarity

适用于：

- 问答任务

- 摘要生成

- 内容改写

# 人工评测（Human Evaluation）

人工评测是保证评测质量的重要手段，尤其适用于难以通过规则计算的复杂任务。

人工评测可以更好地反映：用户体验、内容质量、业务价值、专业准确性。常见方式：

## **专家评测**

由具备专业领域知识的专家进行评估，例如：

- 法律专家评估法律问答准确性

- 医学专家评估医疗建议合理性

- 测试专家评估测试用例质量

**特点：**

- 质量高

- 成本高

- 规模有限

## **众包评测**

通过众包平台或内部标注团队进行大规模人工打分。适用于：

- 可理解性评估

- 满意度评估

- 偏好评估

**特点：**

- 成本相对较低

- 可扩展性好

- 需要设计明确的评分标准

# 裁判模型评测（LLM\-as\-a\-Judge）

裁判模型评测是当前大模型应用中常用的评测方法，即使用一个大模型作为“评委”对另一个模型的输出进行评分。LLM\-as\-a\-Judge 可以评估：

- 回答质量

- 推理过程

- 表达清晰度

- 逻辑一致性

- 是否满足指令要求

**优势**

- 可自动化

- 可规模化

- 成本低于人工评测

- 可覆盖开放式任务

**挑战**

- 模型偏差（bias）

- 稳定性问题

- 评分一致性问题

**常见评测方式**

- 单模型评分（single judge）

- 多模型投票（multi\-judge ensemble）

- pairwise comparison（A/B比较）
