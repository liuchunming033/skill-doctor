## 引言

你写了 Prompt，信心满满地点运行。

第一次：85 分。第二次：72 分。第三次：91 分。

是 Prompt 写错了？还是 LLM 在逗你？

答案：都不是——LLM 本身就是个概率系统，同一个 Prompt 跑三次给三个完全不同的分，这是它的正常行为。你需要的不是更好的 Prompt，是一套方法来管住这个不稳定的裁判。

> **设置 temperature=0 之后分数还会漂移吗？** 设置 temperature=0 之后评分漂移会大幅减少——但其他四种偏差（标准不一致、位置偏见、长度偏见、自我提升）仍然存在，temperature=0 解决不了它们。这就是本章要解决的真正问题。

---

## 💡 五种系统性偏差

你在 Ch8 建了 LLM 裁判，但它会犯五种你不一定察觉的错误：

**1. 评分漂移**

同一条输出，今天评 4 分，明天评 3 分，没有任何变化。根因：LLM 的温度参数让每次采样有随机性。

修复：`temperature=0`，固定 seed。把随机性从评估流程中彻底排除。

**2. 标准不一致**

Rubric 里写"语气自然"——但 LLM 对"自然"的理解会漂移。对第一批样本，4 分是"友好但略显生硬"。对第 100 批样本，4 分可能是"非常流畅"。

修复：Rubric 每一分档都写具体的例子，不用形容词用描述。

**3. 位置偏见**

LLM 偏爱先出现的选项，或偏爱列表中靠前的证据。

用 Ch9 的 Trial 3 举例：客服 Agent 的回复先说了正确的物流信息（5 分），后面才推荐新品（-2 分）。LLM 被前半段吸引，直接给了 5 分，根本没看后半段。**正确的物流信息抢走了 LLM 的注意力。**

修复：对争议样本随机调换评分顺序，看分数是否变化。

**4. 长度偏见**

LLM 认为"更长的回复 = 更好的回复"。

修复：Rubric 中明确写"回复字数对打分无影响"，并加入负例："250 字的冗余回复 = 2 分，不是 4 分"。

**5. 自我提升**

用 GPT-4o 当裁判评估 GPT-4o 生成的输出——它会偏爱自己风格的答案。

修复：用不同模型家族做裁判。Anthropic 的 Claude 评估 OpenAI 的输出，反之亦然。

> **偏差优先级建议**：评分漂移和位置偏见最普遍、影响最大，建议优先排查和修复。其余三种偏差在评分漂移和位置偏见修复后再逐项处理。

## 🔧 校准三步法

发现偏差不是终点，校准才是。

**Step 1：收集人工标注**

拿出 Ch6 的 Ground Truth——这是你唯一的校准基准。你手工标注的 3 条，代表"正确答案长什么样"。

**Step 2：计算 Spearman 相关性**

把 LLM 打分和人工标注放在一起，算相关系数：

```python
from scipy.stats import spearmanr

human_scores = [5, 1, 3]   # Ch6 人工标注（换算为1-5）
llm_scores = [4, 3, 5]     # Ch8 LLM打分

correlation, p_value = spearmanr(human_scores, llm_scores)
print(f"Spearman 相关性: {correlation:.2f}")  # 目标: > 0.8
```

当前案例：相关性 = -0.5，LLM 把 Trial 2（人评不通过）给了 3 分，把 Trial 3（人评部分通过）给了 5 分。严重偏差。

**Step 3：迭代优化 Rubric**

找到偏差来源，修改 Rubric 的描述。再跑一次，再算相关性。重复直到相关性 > 0.8。

**校准目标：Spearman 相关性 > 0.8。低于 0.6，这个裁判不可用。**

## 🎭 多模型对抗

单一裁判有单一偏差。解法：让多个裁判投票。

```python
judges = [claude_judge, gpt4_judge, gemini_judge]
scores = [judge(trace, outcome) for judge in judges]

# 取中位数，抵抗极端值
final_score = statistics.median(scores)

# 或取多数投票
final_pass = sum(s >= 4 for s in scores) >= 2
```

两个要点：

**用不同模型家族**。三个 Claude 投票 ≈ 一个 Claude 投票（偏差相同）。Claude + GPT-4o + Gemini 投票，偏差互相抵消。

**代价是 2-3 倍成本和耗时**。不是每个场景都值得。高风险决策（发布前审查）值得，日常迭代验证（GSB 快速判断）不值得。

**触发条件**：当 Spearman 相关性在 0.6-0.8 之间时考虑多模型对抗来弥补单裁判偏差；低于 0.6 先修 Rubric（校准比对抗更根本）。

## 🎯 本节核心观点

LLM 裁判有五种系统性偏差：

- **评分漂移**：同一答案不同时间得分不同 → temperature=0
- **标准不一致**：Rubric 描述模糊 → 每档写具体例子
- **位置偏见**：偏爱先出现的内容 → 随机调换顺序
- **长度偏见**：更长 = 更好 → Rubric 明确说明
- **自我提升**：偏爱自己风格 → 跨家族裁判

**校准比扩集重要十倍。10 条校准好的规则胜过 100 条未校准的。一致率低于 60%，不要继续扩集，先修 Rubric。**

---

## 实操

对比 Ch6 人工标注和 Ch8 LLM 打分，计算一致率（一致条数 / 总条数）：

1. 如果一致率 < 85%，找出最大的偏差来源——是位置偏见？是 Rubric 描述模糊？还是长度偏好？
2. 针对最大的偏差来源修改 Rubric 的一个维度
3. 重新让 LLM 打分，看一致率有没有提升

这就是 Step 2 的一次完整校准循环。
