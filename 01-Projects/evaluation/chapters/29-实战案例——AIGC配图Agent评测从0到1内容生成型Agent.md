
> 学习目标：通过 AIGC 配图 Agent 评测从 0 到 1 的完整过程，理解内容生成型 Agent 评估的核心挑战——主观标准下的可靠评分，以及 6 步法的实操路径：拆维度→建黄金集→人评+机评→校准→扩集→Bad Case 反推，重点掌握"校准比扩集重要十倍"的工程原则。

---

## 引言

阿里云教你评对话——用 AI 评 AI，抓 Bad Case。

腾讯 TPerf 教你评工具——扣分制 + 0 容忍，守住客观底线。

这一节是最难的一类：**内容生成型 Agent**。

AIGC 配图 Agent 的任务是：给一篇文章自动生成配图。

评估标准？"图好不好看"。

"好不好看"是什么意思？10 个人有 11 个答案。这不是代码规则能判断的，这也不像对话质量那样有相对客观的"功能正确性"做锚点——这是纯主观判断。

那到底怎么评？

AIGC 配图团队给了一个答案：**不是找到绝对标准，而是建立可靠的相对评分体系——核心是校准，不是数量。**

---

## 💡 内容生成型 Agent 的核心挑战

三类 Agent 的评估本质差异：

| 类型 | 标准 | 核心工具 | 最大挑战 |
|------|------|---------|---------|
| 对话型 | 半主观（功能+体验） | LLM 裁判 | 没有唯一正确答案 |
| 工具型 | 客观（步骤+结果） | 代码规则 | 序列比对与容忍度 |
| 内容生成型 | 主观（审美+一致性） | LLM 裁判 + 人工校准 | 评分本身不可信 |

注意最后一行：**评分本身不可信**。

对话型评分不可信时，用多 LLM 对抗来降低偏差。但内容生成型的问题更深——什么是"好的配图"，不同领域（科技文章 vs 情感文章）的标准完全不同，同一张图对不同受众的评分可能相差 2 分（满分 5 分）。

这意味着评分器在未经校准的情况下，根本不能用来做决策。

**校准，是内容生成型 Agent 评估的核心工作。**

---

## 📋 六步法：从 0 到 1 搭起整套评估体系

### Step 1：拆维度（避免"好不好看"这种空泛标准）

直接让 LLM 评"这张图好不好"，结果是不可重复的——不同 prompt、不同时间跑出来的分数差异很大，没有参考价值。

正确做法：把"好不好看"拆成可独立评判的维度。

AIGC 配图团队拆出了四个维度：

```
维度 1：主题相关性  —— 图片内容与文章主题是否匹配？
维度 2：风格一致性  —— 图片风格与文章调性是否一致？（严肃报道 vs 轻松博客）
维度 3：视觉质量    —— 图片构图、清晰度、色彩是否合理？
维度 4：安全合规    —— 是否包含敏感内容？（商业场景的刚需）
```

每个维度独立打分（1-5 分），而不是一个整体分。

**为什么要拆？** 因为 LLM 面对"好不好看"这种整体问题时，会被强势维度（比如视觉质量很高）拖带，忽视弱势维度（比如主题相关性低）。独立评分强制 LLM 对每个维度专注思考。

---

### Step 2：建黄金集（10 个高频场景就能启动）

黄金集（Golden Set）是用来校准评分器的基准数据——每条 case 的"正确分数"经过人工仔细确认，后续所有评分器都对照它校准。

**不要等准备好了再开始——10 个高频场景就能启动。**

AIGC 配图的黄金集选取原则：
1. 覆盖主要内容类型（科技/情感/资讯/教程）
2. 包含正例（明显好的配图）和反例（明显差的配图）
3. 每个 case 只需要：输入（文章摘要）+ 期望风格 + 配图

黄金集不需要标准答案（生成型任务没有唯一标准答案），需要的是：**当你看到这张图时，你的直觉判断是什么，以及这个判断的理由是什么**。

这个"理由"就是后续校准 LLM 裁判的材料。

---

### Step 3：人评 + 机评（选择打分体系）

黄金集建好后，同时运行人工评分和 LLM 评分，建立初始对照。

打分体系选择（回顾 Ch9）：
- **GSB（Good/Same/Bad）**：适合横向比较（这次生成的图比上次好吗？）
- **MOS（整体均分）**：适合跨时间的趋势追踪（这周的配图整体质量上升了吗？）
- **Likert（逐维度打分）**：适合精细诊断（到底哪个维度拖了后腿？）

AIGC 配图团队主要用 Likert，因为他们需要精确知道是哪个维度的问题，才能定向优化生成 Prompt。

**评分 Prompt 的强制约束：**

```
你是一位专业的图片评审编辑。请按以下步骤评估这张配图：

1. 主题相关性（1-5 分）：[独立评估 + 理由 + 引用文章中的具体内容]
2. 风格一致性（1-5 分）：[独立评估 + 理由 + 引用图片中的具体视觉元素]
3. 视觉质量（1-5 分）：[独立评估 + 理由]
4. 安全合规（通过/不通过）：[是否包含敏感内容]

输出格式（JSON，不允许其他格式）：
{
  "relevance": {"score": int, "reason": str, "evidence": str},
  "style": {"score": int, "reason": str, "evidence": str},
  "visual": {"score": int, "reason": str, "evidence": str},
  "safety": {"pass": bool, "issue": str | null}
}
```

**"evidence"字段是强制的**——强迫 LLM 引用具体内容作为打分依据，而不是空泛结论。这是提高评分可信度最有效的单一手段。

---

### Step 4：校准 AI 评分（校准比扩集重要十倍）

把 LLM 评分结果和人工标注对比，计算三个对齐指标：

```
指标 1：各维度分差
  人评风格一致性均分：3.8，LLM 评：4.6 → 分差 0.8
  判断标准：分差 > 1.0 = 对齐不足，需要调整

指标 2：排序倒置率
  人工认为 A > B，LLM 认为 B > A 的比例
  判断标准：倒置率 > 20% = 对齐不足

指标 3：极端 case 一致率
  人评最高分（5）的 case，LLM 也评高分（4-5）的比例
  判断标准：一致率 < 80% = 对齐不足
```

AIGC 配图团队的实际校准过程：

**第一轮**：LLM 的风格一致性维度分差 = 0.8（可接受范围），但排序倒置率 = 35%（不对齐）。

**问题定位**：LLM 在科技类文章和情感类文章上用了同一个"一致性"标准——科技文章配一张有数据图表的图，LLM 打高分；情感文章配同一张图，LLM 也打高分。但人工评分认为情感文章配数据图表是不合适的。

**调整**：在 Rubric 中增加"先判断文章类型，再用对应类型的一致性标准评估"。

**第二轮**：排序倒置率从 35% 降至 12%，对齐达标。

这个过程：从发现问题到校准完成，花了 2 天。但它让后续所有 LLM 评分的可信度提升了一倍。

**为什么"校准比扩集重要十倍"：**

没有校准的评分器，覆盖 500 条样本也是无效的——你收集的是噪声，不是信号。

校准好的评分器，即使只有 10 条黄金样本，每次打出来的分数都是可信的判断依据。

**不要在不可信的评分器上扩规模。**

---

### Step 5：扩集到生产规模

校准完成后，开始扩大评测规模：

```
黄金集（10 条）→ 用于校准
生产集（100-500 条）→ 按用户 query 频率构建，日常评估用

生产集的人工介入比例：
- 5% 样本人工抽检（校准维护）
- 95% 全部 LLM 自动打分
```

**按 query 频率构建**的意思是：哪类配图请求最多，生产集中就要有更多这类样本。

如果 80% 的请求是科技文章配图，生产集中科技文章类 case 也应该占多数——这样生产集的分数才能真实反映整体服务质量，而不是被低频场景的高分拉高平均值。

---

### Step 6：Bad Case 反推 + 动态进化

生产集跑起来后，持续做 Bad Case 反推（回顾 Ch26）。

AIGC 配图团队的一个具体例子：

**现象**：科技类文章中，"概念解释类"文章（比如"什么是量子计算"）的配图主题相关性评分长期偏低。

**归因**：不是生成的图本身差——是生成 prompt 里缺少对"抽象概念的隐喻化视觉表达"的指导。Agent 生成的 prompt 倾向于字面匹配（"量子"→用量子计算机图片），而不是概念隐喻（"量子叠加态"→用薛定谔猫或多重可能性的图片）。

**优化**：在文章类型识别逻辑里，增加"概念解释类"的识别，以及对应的隐喻化 prompt 生成策略。

**效果**：概念类文章的主题相关性评分从平均 2.9 提升到 4.1（满分 5）。

**动态进化机制**：

```
黄金集  → 每月更新一次（纳入新发现的典型 case）
生产集  → 新模型上线后，回跑全部历史 case（而不是只跑新 case）
评分器  → 随 Rubric 调整，对历史 Trace 重新评分（参考 Ch28 的重新评分模式）
终极目标 → 系统能自动识别"什么类型的文章+什么类型的配图=系统性低分"，触发自动优化
```

## 三类 Agent 评估横向对比

三章结束了，回顾一下三类 Agent 的评估核心差异：

| 维度 | 对话型（阿里云） | 工具型（腾讯 TPerf） | 内容生成型（AIGC 配图） |
|------|----------------|---------------------|----------------------|
| 标准类型 | 半主观 | 客观 | 纯主观 |
| 核心评分器 | LLM 裁判 | 代码规则 + LCS | LLM 裁判（重度校准） |
| 稳定性策略 | 多 LLM 对抗 | 0 容忍 | 校准驱动 |
| 最大投入 | 场景映射 + 上下文工程 | 步骤序列定义 | 校准（持续维护） |
| 核心洞察 | 评估-诊断-优化三段闭环 | 扣分制暴露薄弱点 | 校准比扩集重要十倍 |

## 🎯 本节核心观点

**内容生成型 Agent 评估的核心挑战：评分本身不可信——校准是前提，不是优化。**

**六步法：拆维度→建黄金集（10 条启动）→人评+机评→校准→扩集→Bad Case 反推**

**三个对齐指标（判断 LLM 评分器是否可信）：**
1. 各维度分差 < 1.0
2. 排序倒置率 < 20%
3. 极端 case 一致率 > 80%

**动态进化机制：黄金集每月更新 + 新模型回跑历史 + 评分规则独立迭代**

**核心原则：校准比扩集重要十倍。10 条校准好的黄金样本，比 500 条未校准的生产样本更有价值。**

## 实操

**第一步**：参照六步法，为你自己的 Agent 写一份"从 0 到 1 搭建评测系统"的执行计划。每一步列出：
- 具体产出（例：Step 1 = 4 个独立可评分的维度定义文档）
- 完成时间节点
- 判断该步完成的标准

**第二步**：针对 Step 4（校准），立刻可以做的最小动作是：
- 用你 Ch8 设计的评分 Rubric，对同一批样本（Ch6 的 3 条手工标注）跑一次 LLM 评分
- 计算分差：LLM 的平均分和你的人工标注均分差了多少？
- 如果分差 > 1.0，找出最大的分歧点，调整 Rubric

**第三步**：你的评分 Prompt 里有 evidence 字段吗？如果没有，立刻加上——这是提高评分可信度最简单的单一改进。
