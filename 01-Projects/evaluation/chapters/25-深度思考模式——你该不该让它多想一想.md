
> 学习目标：理解深度思考模式与普通 CoT 的本质区别，掌握"该不该开"的判断框架，学会用准确度/耗时/成本三角衡量深度思考的投入产出比——不是每个评估任务都值得"多想一想"。

---

## 引言

深度思考模式就像让裁判在打分前先自言自语十分钟。有时这十分钟值回票价，有时纯属浪费。

怎么判断？

很多工程师的默认选择是"开就开吧，更准一点总没错"。错了。深度思考不只是"更慢一点"，它是慢数倍、贵数倍——如果你有 500 条对话要评估，深度思考可以把成本和时间都乘以 10。

所以问题不是"深度思考好不好"，而是"这个任务值不值得"。

## 💡 深度思考 ≠ 普通 CoT

先搞清楚这两件事的区别，才能知道什么时候该用哪个。

**普通 CoT（Chain of Thought）**

你在 Prompt 里写："请一步一步思考，然后给出答案。"LLM 按照你的指示，把推理过程写出来。

这是你控制的外显推理——LLM 输出了推理链，你能看到它怎么想的。

**深度思考模式（Extended Thinking）**

不是你在 Prompt 里加一句话。这是一个模型层面的开关——模型在生成最终回答之前，有一个独立的"思考阶段"，进行多轮内部推理，对自己的推理过程做自我批判和验证。

深度思考的五步内部流程：
1. **问题解析**：把问题拆解成子问题
2. **多路径推理**：同时考虑多种可能的判断方向
3. **逐步验证**：对每一条推理路径做正确性检查
4. **自我批判**：主动寻找自己推理中的漏洞
5. **最终整合**：综合多路径结论，输出最终判断

这比普通 CoT 深得多。也贵得多——思考过程本身也消耗 Token，而且不会出现在你的最终输出里，你只看到结论，却已经付了思考的费用。

## 🔧 两个任务的对比实验

同一个客服 Agent 评估场景，两类任务：

**任务 A：退款操作检测**

```
任务描述：这条对话里，Agent 有没有执行退款操作？
评估维度：是/否的二元判断
```

代码评分器：检查 Trace 里有没有 refund_api 的调用记录。精确，0 误差。

如果用代码能解决，开深度思考就是浪费。这种"判断是什么"的任务，LLM 裁判本身都是多余的——更别说深度思考了。

**任务 B：回复得体性评估**

```
任务描述：用户情绪激动、语气强烈，Agent 在这个时机的回复——
          是否得体？是否在正确的时间说了正确的话？
评估维度：多维度语义理解 + 语境判断 + 情感敏感性
```

关闭深度思考：评分与人工评分的 Spearman 相关性 = 0.65

开启深度思考：Spearman 相关性 = 0.82

提升了 0.17。这个任务值得多花 10 倍的时间和成本。

## 🎭 判断框架：开还是不开？

核心问题只有一个：

**这个任务需要"理解为什么"，还是只需要"判断是什么"？**

| 任务类型 | 深度思考 | 理由 |
|---------|---------|------|
| 字段存在性检查 | ❌ 关闭 | 代码规则能解决，不需要 LLM |
| 格式合法性验证 | ❌ 关闭 | 代码规则能解决，不需要 LLM |
| 事实查询匹配 | ❌ 关闭 | 精确匹配，无需推理 |
| 情感得体性判断 | ✅ 开启 | 需要多维度语境理解 |
| 边界道德判断 | ✅ 开启 | 需要权衡多个视角 |
| 复杂意图识别 | ✅ 开启 | 需要理解潜台词和隐含意图 |
| 风险行为识别 | ✅ 开启 | 需要推理行为后果 |

一个简单的判断规则：

如果你能写出精确的规则或代码来判断 → 关闭深度思考（用代码）

如果人工标注者也需要"想一想"才能判断 → 开启深度思考

## ⚠️ 深度思考的代价

**时间代价**

响应时间通常慢 3-10 倍。500 条评估任务，原来 10 分钟跑完，开深度思考可能需要 1-2 小时。

**成本代价**

思考过程消耗的 Token 也计费（即使你看不到这些 Token）。实际成本通常是 3-10 倍。

**TPM 配额代价**

思考 Token 也占用 TPM（每分钟 Token 限额）。高并发评估时，深度思考容易把你的配额打满，导致限速。

**这三个代价叠加的结果**：如果你的评估任务 80% 是"判断是什么"类型，全开深度思考，你在把 80% 的成本花在不需要深度思考的地方。

**正确策略：混合模式**

```
代码评分器  ← 精确匹配任务（字段/格式/事实）
LLM 标准模式 ← 需要语义理解但不复杂的任务
LLM 深度思考 ← 需要多维语境判断的任务（占 20%-30%）
```

## 🎯 本节核心观点

**深度思考 ≠ 普通 CoT：**
- CoT 是你控制的外显推理
- 深度思考是模型内部的五步自我批判循环（解析→多路径→验证→批判→整合）

**开启条件：**
- 需要"理解为什么"→ 开
- 只需要"判断是什么"→ 关

**代价三角：时间 × 成本 × TPM 配额**——全部慢/贵/占用多

**实践原则：分层使用。** 代码解决的不用 LLM，LLM 能解决的不开深度思考，只有真正需要多维推理的任务才值得"多想一想"。

**一句话判断：如果人工标注者也需要"想一想"，就值得开。如果答案一眼就能看出来，就是浪费。**

---

## 实操

对同一条评估任务，做一次 A/B 对比：

**第一步**：选一条评估任务（优先选"回复得体性"或"意图识别准确性"类型的，而非字段检查类）。

**第二步**：关闭深度思考，跑 5 次，记录：
- 平均评分
- 与你的人工判断一致的次数
- 平均耗时

**第三步**：开启深度思考，同一条任务跑 5 次，记录相同指标。

**第四步**：计算这个任务值不值得开：准确度提升多少？成本增加了多少倍？你的任务量乘以成本增量，可接受吗？

## 下节预告

深度思考解决了"判断质量"的问题。但还有一个问题没解决：同一个任务跑 10 次，成功 7 次——这 70% 算过关吗？非确定性场景下，你需要统计证据，而不是一次结论。
