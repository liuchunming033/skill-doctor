# 详细检查清单

## 1. description检查清单

**检查项**：
- [ ] 是否包含触发关键词（"当用户说"、"在XXX场景时触发"）
- [ ] 是否避免功能说明表述（"这是一个"、"该Skill用于"）
- [ ] 长度是否合理（<100字符）
- [ ] 是否包含用户会说的关键词
- [ ] 是否说明适用场景

**评分标准**：
- 90分：完美符合（触发条件明确、关键词完整）
- 70分：基本符合（有触发关键词但不够精准）
- 50分：不符合（写成功能说明）

---

## 2. gotchas检查清单

**检查项**：
- [ ] gotchas.md是否存在
- [ ] 内容是否非空（>500字节）
- [ ] 是否有"现象-问题-解决"结构
- [ ] 是否有真实坑点（不是"暂无"、"TODO")
- [ ] 坑点数量是否合理（>3个）

**评分标准**：
- 90分：优秀（>5个真实坑点，结构完整）
- 70分：良好（有坑点但内容较少）
- 40分：不合格（空壳或占位符）

---

## 3. 文件组织检查清单

**检查项**：
- [ ] SKILL.md主文件是否<200行
- [ ] 是否有附属文件（references/或同级.md文件）
- [ ] 是否使用[链接](xxx.md)引用
- [ ] 目录结构是否合理
- [ ] 是否遵循渐进式披露原则

**评分标准**：
- 90分：优秀（主文件<150行，附属文件完整）
- 70分：良好（主文件<250行，有部分拆分）
- 50分：不合格（单文件堆砌>300行）

---

## 4. 过度约束检查清单

**检查项**：
- [ ] 是否避免"第一步第二步"表述
- [ ] 是否有决策点（"判断"、"选择"、"根据"）
- [ ] 是否约束目标而不是路径
- [ ] 是否给Agent足够自由度
- [ ] 是否描述决策框架

**评分标准**：
- 90分：优秀（决策框架清晰，无过度约束）
- 70分：良好（有决策点但有部分路径约束）
- 50分：不合格（死板流程，无决策点）

---

## 5. 已知知识检查清单

**检查项**：
- [ ] 是否避免"什么是"、"介绍"等教程式表述
- [ ] 是否补充团队特有知识
- [ ] 是否避免重复Agent已知内容
- [ ] 是否聚焦Agent不知道的部分
- [ ] 是否有价值增量

**评分标准**：
- 90分：优秀（只补充Agent不知道的）
- 70分：良好（大部分是特有知识）
- 40分：不合格（大量教程式内容）

---

## 6. 内存配置检查清单

**检查项**：
- [ ] config.json是否存在
- [ ] 配置是否合理（不是空文件）
- [ ] 是否包含团队偏好
- [ ] Skill是否需要持久化配置
- [ ] 配置项是否有说明

**评分标准**：
- 90分：优秀（config完整且合理）
- 70分：良好（有配置但较简单）
- 50分：不合格（无config.json）
- N/A：Skill不需要配置（不扣分）

---

## 7. 脚本检查清单

**检查项**：
- [ ] Skill是否需要脚本（稳定能力）
- [ ] scripts目录是否存在
- [ ] 脚本是否有注释
- [ ] 脚本是否有错误处理
- [ ] 脚本调用方式是否正确

**评分标准**：
- 90分：优秀（脚本完整、注释清晰）
- 70分：良好（有脚本但较简单）
- 50分：不合格（需要脚本但缺失）
- N/A：Skill不需要脚本（不扣分）

---

## 8. Hooks检查清单

**检查项**：
- [ ] hooks配置格式是否正确
- [ ] matcher是否正确（"Bash"、"Skill"、"Read"等）
- [ ] command路径是否正确
- [ ] 是否有PreToolUse/PostToolUse类型标注
- [ ] hook脚本是否可执行

**评分标准**：
- 90分：优秀（hooks配置完美）
- 70分：良好（hooks存在但有轻微问题）
- 50分：不合格（hooks格式错误）
- N/A：Skill不需要hooks（不扣分）

---

## 9. Markdown格式检查清单

**检查项**：
- [ ] YAML frontmatter是否存在
- [ ] YAML是否包含name和description
- [ ] 表格是否对齐
- [ ] 代码块是否标注语言
- [ ] 列表是否格式正确

**评分标准**：
- 90分：优秀（格式完全规范）
- 70分：良好（有轻微格式问题）
- 50分：不合格（YAML缺失或格式混乱）

---

## 10. 综合检查清单

**检查项**：
- [ ] 是否有下节预告（如果是课程类）
- [ ] 是否有参考资料
- [ ] 是否使用[[wikilink]]链接
- [ ] 是否有学习目标
- [ ] 引言是否口语化

**评分标准**：
- 90分：优秀（结构完整、链接清晰）
- 70分：良好（有大部分元素）
- 50分：不合格（缺少关键元素）

---

## 总评分公式

```
总分 = (检查项1得分 + 检查项2得分 + ... + 检查项10得分) / 10
等级 = A(90-100) / B(75-89) / C(60-74) / D(<60)
```

**权重调整**：
- description：权重1.5（最重要）
- gotchas：权重1.5（最有价值）
- 其他项：权重1.0
