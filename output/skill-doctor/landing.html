// 01. Diagnostics

九大体检项目

从 description 触发条件到 markdown 格式规范，全面覆盖 Skill 质量维度。

🔎

Description 检查

是否写成触发条件而非功能说明，长度 <100 字符

⚠️

Gotchas 检查

坑点记录是否存在且充实，"现象-问题-解决"结构完整

🗃️

文件组织检查

三分类标准 + 轻量 Skill 豁免，参考材料不放根目录

🏃

过度约束检查

是否包含死板步骤，有无决策点框架

📚

已知知识检查

是否重复 Agent 已知内容，是否补充团队特有知识

⚙️

内存 / 脚本 / Hooks

config.json 持久化、脚本封装、Hooks 安全防护

🖌️

Markdown 格式

YAML frontmatter、表格对齐、代码块语言标注

⚡

轻量 Skill 豁免

单文件 ≤80 行自动判定合理，不强制要求拆分

📈

收益估算

Token 节省 + 耗时浪费统计，优化 ROI 一目了然

// 04. Workflow

三步完成体检

脚本负责稳定层（评分计算），Agent 负责智能层（优化建议、交互、执行修改）。

1. 运行诊断脚本

python scripts/analyze_skill.py <path> --output json 自动扫描九大维度，生成结构化报告。

2. 选择优化项

Agent 补充具体优化建议 + 预期提分 + 工作量评估，你勾选要修复的项。

3. 自动修复 & 对比

Agent 直接修改文件，修复后重新体检，输出前后对比报告。
