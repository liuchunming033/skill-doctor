## 核心摘要

Claude Code 核心团队盘点内部几百个 Skill，总结为 9 类能力地图，收束到四个环节：认知→生产→验证→交付。认知环节：内部库与接口文档、数据获取与分析、故障排查手册；生产环节：代码脚手架与模板、业务流程与团队自动化；验证环节：代码质量与审查、产品验证；交付环节：持续集成与部署、基础设施运维。这张地图帮团队看清哪些能力已沉淀成系统，哪些关键经验还停留在人脑里，真正短板在于关键经验有没有被做成可调用、执行、验证和复用的 Skill。

## 关键洞察

### 核心问题

**AI 放大能力缺口**：
- AI 会放大组织已有的能力，也会放大能力缺口
- 如果关键经验还没有 Skill 化，AI 产出速度越快，组织混乱程度也越快
- **真正短板**：关键经验有没有被做成可调用、执行、验证和复用的 Skill

### 四个环节九类 Skill

#### 第一环节：认知

**解决**：Claude 到底知不知道自己面对的是什么系统？从哪里获得理解？遇到异常时从哪里开始查？

**1. 内部库与接口文档**
- 补充组织内部的专属知识（内部库、内部平台、内部命令行工具、内部设计系统）
- **最有价值内容**：最常用的调用方式、参考片段、文档里没写但团队反复踩坑的 gotchas
- **原则**：不要写 Claude 本来就知道的东西，要写它推不出来的东西

**2. 数据获取与分析**
- 解决分析任务空转的问题（数据入口）
- 把数据源凭证、仪表板 ID、常用查询语句、分析脚本整理成 Claude 能直接调用的东西
- 示例：漏斗分析、分群、对比、监控面板接入

**3. 故障排查手册**
- 接收症状（Slack 告警、错误签名、请求 ID），自动查日志、查监控、串链路
- 输出结构化排查结果
- **价值**：承接低频但认知负担极高的问题，把资深工程师排障思路变成组织可复用的基础设施

#### 第二环节：生产

**解决**：Claude 怎么稳定地产出代码、流程结果和标准化产物，而不是每次从零起步。

**4. 代码脚手架与模板**
- 给 Claude 一套稳定的起手式（新建前端组件、新开接口、新起数据库迁移、新建工作流、新拉微服务模板）
- **问题**：没有这类 Skill，Claude 每次重新决定目录结构、命名习惯、文件分层，产出容易漂

**5. 业务流程与团队自动化**
- 处理高频重复、规则清楚的流程工作
- 示例：站会汇报（汇总任务系统、代码仓库和聊天工具历史）、事故报告（自动创建报告、同步状态、通知相关人）
- **价值**：持续吃掉团队低价值、重复性劳动，最容易被快速感知收益

#### 第三环节：验证

**解决**：Claude 能不能证明自己写出来的东西真的能工作。

**6. 代码质量与审查**
- 检查代码、文档或方案质量（安全风险、结构问题、命名习惯、性能隐患、团队规范）
- **Fresh eye review**：制造新鲜视角专门挑毛病，对抗模型的自我一致性偏差
- 不要只让 Claude 用刚写完代码的视角回头审自己

**7. 产品验证**
- 配合无头浏览器、TMUX、脚本和断言测试代码是否真的正确工作
- **重要技巧**：让 Claude 录制输出视频（看到完整执行过程），编程式断言强制检查状态
- **核心**：让 Claude 有能力证明自己做对了，而不只是交差
- **价值**：验证类 Skill 值得工程师花一周时间专门打磨，真正决定质量的不只是生成能力，还有反馈闭环

#### 第四环节：交付

**解决**：Claude 怎么把结果真正推进到系统里，并在后续阶段持续维护。

**8. 持续集成与部署**
- 负责 PR 监控、合并冲突处理、条件满足后自动合并
- 负责服务部署、灰度发布、盯关键指标、异常时自动回滚
- **问题**：很多团队"一到部署就必须完全切回人工"，说明 Agent 能力链条只走到一半

**9. 基础设施运维**
- 处理基础设施和日常运维动作（高风险动作）
- 示例：查孤立资源、处理依赖升级、调查云资源费用异常、容器集群故障处理、执行待确认点的清理流程
- **特别强调护栏**：删资源、改环境、切流量这些事一旦失控代价很高，先把风险边界写清楚，再谈自动化

### 组织能力地图

| 环节 | 阶段任务 | Skill 类别 |
|------|---------|-----------|
| **认知** | 让 Claude 先理解系统、接上数据、掌握排障入口 | 内部库文档、数据分析、故障排查 |
| **生产** | 让 Claude 稳定地产出代码和流程结果 | 代码脚手架、业务流程自动化 |
| **验证** | 让 Claude 不只是产出，还能做质量审查、完成产品验证 | 代码质量审查、产品验证 |
| **交付** | 把结果推进到代码合并、部署和后续运维 | CI/CD、基础设施运维 |

### 如何使用这张地图

把自己团队现有的 Skill 摆上去，很快就能看出来：
- 哪些地方已经沉淀下来了
- 哪些关键环节还停留在人脑里
- 哪些地方明明最需要 Skill，结果到现在还没被系统化

## 核心思想

**团队落地 AI 研发时，真正的短板往往在于关键经验有没有被做成 Claude 能真正调用、执行、验证和复用的 Skill。**
