## 核心摘要

以飞书项目为流程枢纽，构建 AI 驱动的产品需求自动化流水线：用户输入一句话需求，系统依次自动生成 Spec/PRD、技术方案、测试用例，每个节点均设置 AI 自动质检 + 人工评审双重门禁，评审通过后产物自动回流至 Obsidian 知识库归档。

## 五节点流水线

| 节点             | 动作                                                             | 负责方  |
| ---------------- | ---------------------------------------------------------------- | ------- |
| 1. 创建需求      | 飞书项目新建 Issue，自动生成唯一 RID                             | 人工    |
| 2. 生成 Spec/PRD | Agent 从 Obsidian 全库召回信息，生成标准 PRD+HTML 原型           | Agent   |
| 3. 生成技术方案  | Agent 结合 Spec 产物，生成技术方案文档                           | Agent   |
| 4. 生成测试用例  | Agent 结合前序产物，生成全套测试用例                             | Agent   |
| 5. 门禁+评审     | AI 一致性校验 + 人工评审 → 通过则归档，不通过则 Agent 迭代重生成 | AI+人工 |

## 核心优势

1. **严格技能依赖**：流水线可控、可溯源，不会乱生成
2. **人机精准分工**：人只做创意与决策，AI 承担全部结构化产出
3. **全链路可追溯**：原始素材 → 知识依据 → 产物 → 评审全程闭环
4. **全流程可度量**：各环节状态自动流转，耗时和 AI 成本（Token）可度量
