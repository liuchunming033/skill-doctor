## 定义

渐进式披露（Progressive Disclosure）是 Agent Skill 的核心加载机制，通过三层结构（元数据层→正文层→附属文件层）按需加载技能内容，让 Claude 拥有近似无限上下文的工作方式。类比：先读目录确定主题，再读章节掌握要点，最后按需查阅附录。

## 三层加载架构

### Level 1：YAML 元数据层

- **文件位置**：`SKILL.md` 顶部 YAML front matter
- **内容**：技能 name + description
- **加载方式**：始终加载（Agent 启动时预加载）
- **Token 消耗**：约 100 tokens
- **作用**：告诉模型有哪些技能以及它们做什么
- **设计目标**：索引而不展开

### Level 2：正文内容层

- **文件位置**：`SKILL.md` 的 Markdown 正文部分
- **内容**：技能的核心指令与 SOP 逻辑
- **加载方式**：技能被触发时加载
- **Token 消耗**：平均 < 5000 tokens
- **作用**：当 Claude 决定使用技能时，打开这部分内容指导执行

### Level 3 及以上：附属文件层

- **文件位置**：技能文件夹中除 `SKILL.md` 外的其他文件
  - Markdown 文档：`forms.md`、`reference.md`
  - 配置文件：`.json`
  - 脚本文件：`.py`
- **加载方式**：按需加载（SOP 提到时才读取/执行）
- **Token 消耗**：理论上无限
- **作用**：支持复杂任务的详细文档、配置和脚本

## 核心优势

### 突破上下文窗口限制

| 传统方式 | 渐进式披露 |
|---------|----------|
| 所有知识固定在系统提示 | 按需加载相关技能 |
| 受上下文窗口限制 | 理论上无限扩展 |
| 粗粒度，包含所有知识 | 细粒度，按场景精准加载 |

### 实现近似无限上下文

- **元数据常驻**：所有技能的 name+description 始终在上下文中（每个技能 ~100 tokens）
- **正文按需**：只加载被触发技能的正文（< 5000 tokens）
- **文件按需**：只读取 SOP 提到的附属文件
- **脚本外部执行**：脚本代码不进入上下文，只有结果返回

## 技能触发动态流程

1. **初始状态**：上下文 = 系统提示 + 所有技能元数据 + 用户输入
2. **判断触发**：Claude 判断任务与某技能相关
3. **读取正文**：调用工具读取该技能的 `SKILL.md` 正文（Level 2）
4. **按需展开**：正文中提到的附属文件（Level 3）才被读取
5. **执行脚本**：SOP 要求运行脚本时，在虚拟机执行，只返回结果
6. **完成任务**：综合上下文内容生成最终响应

## 类比理解

### 类比书籍阅读

- **Level 1** = 目录：快速了解书籍包含哪些章节
- **Level 2** = 章节：阅读感兴趣的章节内容
- **Level 3** = 附录：按需查阅详细的参考资料

### 类比操作系统

- **传统方式** = 把所有程序都加载到内存
- **渐进式披露** = 按需加载程序到内存，其他程序留在硬盘

## 设计哲学

**近似无限上下文不是模型参数的魔法，而是一种架构思想**——让模型的上下文只装载需要思考的那一页，而把真正的世界留在它随时可访问的文件系统里。
