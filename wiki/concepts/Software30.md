## 定义

Software 3.0 是由 Andrej Karpathy 命名的新软件开发时代：软件不再由人类手写逻辑，而是通过自然语言（Prompt）描述意图，由 LLM 执行实现。核心变革是 LLM 成为新的运行时，人类从"程序员"转变为"需求表达者 + 决策评审者"。

## 三个时代对比

| 时代         | 核心范式                   | 主要角色            |
| ------------ | -------------------------- | ------------------- |
| Software 1.0 | 人类手写代码，计算机执行   | 程序员              |
| Software 2.0 | 神经网络权重，梯度下降训练 | 机器学习工程师      |
| Software 3.0 | 自然语言 Prompt，LLM 执行  | 产品/业务人员 + LLM |

## Software 3.0 时代企业研发的四大特征

1. **AI Native**：AI 是研发流程中的一等公民，不是辅助工具
2. **上下文驱动**：产出质量由上下文（知识库）质量决定，而非个人技能
3. **流程标准化**：需求→设计→开发→测试的每个节点都有 Agent 介入
4. **资产积累型**：每次交付都沉淀到知识库，形成持续复利

## 对企业的冲击

- 技术壁垒从"谁会写代码"转向"谁有更完整的领域知识库"
- 个人效率差距缩小，组织知识沉淀成为核心竞争力
- 知识库质量 = 产出质量上限（"垃圾进，垃圾出"）

## 知识库建设专项

针对企业 12 条产品线的 Software 3.0 落地专项，目标是：

- 四维完整知识体系（产品 / 设计 / 技术 / 测试）
- 标准化三类核心产物（SPEC / 技术方案 / 测试用例）
- 识别企业级知识中心建设需求
