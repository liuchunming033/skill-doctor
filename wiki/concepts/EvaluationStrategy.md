## 定义
Agent 评估策略是指在质量保障体系中，根据不同目标选择和组织评测方法的组合方案。核心包括四大维度：端到端 vs 分层评测、线下 vs 线上评测、打分 vs 对比评估、以及评估→归因→优化的闭环策略。

## 维度一：端到端评测 vs 分层评测

| 维度 | 端到端评测（E2E） | 分层评测（Layered） |
|------|-----------------|-------------------|
| **视角** | 黑盒，模拟真实用户场景 | 白盒，排除上下游干扰 |
| **关注点** | 最终任务是否成功、用户满意度 | 具体算子/环节的表现 |
| **作用** | 准入验证——"能否发布？" | 缺陷定位——"该修哪里？" |
| **成本** | 高（全链路运行） | 低（单环节快速迭代） |

**组合逻辑**：端到端评测发现问题，分层评测定位并解决问题。

分层可按三个维度进行：
- **按架构逻辑分层**：意图理解层(S1)→信息检索层(S2)→逻辑规划层(S3)→内容包装层(S4)→UI交互层(S5)
- **按评估粒度分层**：系统效率层（延迟/Token成本）→会话级别（任务成功率/轨迹质量）→节点级别（工具选择/参数精度）
- **按业务维度分层**：业务场景→具体任务

## 维度二：线下评测 vs 线上评测

| 维度 | 线下评测（Offline） | 线上评测（Online） |
|------|-------------------|-------------------|
| **类比** | 模拟考试 | 实战监测 |
| **阶段** | Agent 发布前 | Agent 部署后 |
| **数据** | 黄金数据集（专家策划+Ground Truth） | 生产日志、实时交互轨迹 |
| **目的** | 准入验证、回归保护、安全/红队测试 | 监测预警、链路追溯、效果闭环 |
| **手段** | 模拟器多轮对话、LLM-as-Judge 自动打分 | OpenTelemetry 链路采集、用户反馈收集 |

**关键闭环**：线上发现的失败案例回流至线下扩充黄金数据集，是提升 Agent 效果的核心路径。

## 维度三：打分评估 vs 对比评估

| 维度 | 打分评估（Absolute Scoring） | 对比评估（Pair-wise / Relative） |
|------|---------------------------|-------------------------------|
| **类比** | 体检报告 | 运动员排名 |
| **参照系** | 绝对标准、预设KPI、历史自我 | 其他竞争对象（竞品/替代方案） |
| **输出** | 二元化（通过/不通过）或绝对分值 | 排序（A>B>C）、相对差距、GSB结论 |
| **目的** | 准入验证——"能否发布？" | 选型决策——"用A还是B？" |
| **数据要求** | 仅需待评对象+明确阈值 | 必须多对象同条件数据 |

## 维度四：闭环策略（Evaluation → Diagnosis → Optimization）

```
端到端评测 → 获得整体基线
    ↓
失败案例归因 → 按架构层级分类失败原因
    ↓
分层专项评测 → 对薄弱环节构建黄金数据集快速迭代
    ↓
优化实施 → Prompt/工具/知识/模型针对性改进
    ↓
二次端到端验证 → 确认局部改进转化为整体提升
    ↓
持续闭环 → 线上新问题回流线下评测集
```

核心思想：**评测不是终点，优化才是目的；闭环策略让每一次评测都能真正推动系统能力提升。**
