## 定义
Agent 效果（Agent Effect）是对智能体在执行任务、决策制定和用户交互过程中表现的综合评估与理解。它不仅停留在表面的文本生成质量，更关注智能体的整体行为、任务成功率以及与用户意图的一致性。

## 影响 Agent 效果的关键因素

| 维度 | 具体因素 |
|------|---------|
| **核心能力组件** | 自主决策与多步推理、工具调用准确性、长程上下文记忆利用、复杂目标分解规划、基于反馈的自我反思与修正 |
| **数据与环境** | 评测集的生产保真度（是否具有真实用户数据特征）、任务复杂度、与外部系统交互的稳定性 |

## 评测目的

1. **技术与业务验证**：及时发现决策偏差避免损失（如金融风控），验证是否满足业务需求提升效率
2. **驱动迭代优化**：分析评估数据定位薄弱环节（意图理解 vs 工具调用），为模型和工程优化提供方向
3. **伦理与安全合规**：排查偏见、歧视、数据隐私泄露或违反法律法规的风险

## 评测难点

| 难点 | 说明 |
|------|------|
| **行为不确定性** | Agent 具有非确定性特征，复杂任务导致行为难以预测 |
| **传统指标局限** | BLEU、ROUGE 等传统文本指标无法捕捉软件质量的微妙方面 |
| **LLM-as-Judge 风险** | 虽成主流但存在自身偏见和对抗性操纵（评分通胀攻击）风险 |
| **数据获取成本高** | 高质量黄金数据集获取成本高昂，需要领域专家参与标注 |
| **故障定位复杂** | 多步工作流中误差累积，下游错误由上游组件引起，根因定位困难 |

## 与传统软件测试的本质区别

| 维度 | 传统软件 QA | Agent 评测 |
|------|-----------|-----------|
| **输出确定性** | 固定输入→固定输出 | 相同输入可能产生不同输出 |
| **评估维度** | 功能正确性为主 | 多维度：正确性、效率、安全性、体验 |
| **测试方式** | 单元/集成/E2E 测试 | 端到端+分层+线上线下组合 |
| **评估标准** | 通过/失败二元判定 | 多层次打分+对比+归因闭环 |

Agent 评测已从传统的单一"通过/失败"测试，演变为一种**面向效果、层次化（系统级、会话级、节点级）且持续迭代的科学评估体系**。
