## 定义

Inversion 模式（先问清需求再开工）是 Google 总结的五种 Skill 设计模式之一，解决"信息不全却急着开工"的问题。核心做法：把流程反过来——Agent 先发问，用户先回答。三阶段：Discovery（先确认你到底要解决什么问题）→ Constraints（把平台、技术栈、限制条件、交付要求问完整）→ Synthesis（信息齐了之后才允许开始生成）。用门控指令明确规定——信息没齐不准开工。

## 解决的问题

**信息不全却急着开工**：
- Agent 太爱猜，一拿到模糊需求就急着生成
- 信息不够就自己补，而且补得非常自信
- 导致输出偏离真实需求，需要反复修改

**根本原因**：
- Agent 缺少信息澄清机制
- 模型倾向于"自信猜测"而非"主动询问"
- 缺少门控约束阻止过早开工

## 核心做法

### 三阶段流程

```
Phase 1: Discovery (问题确认)
├── 确认核心问题：你到底要解决什么问题？
├── 明确目标：期望的输出是什么？
└── 理解背景：这个任务的背景和上下文是什么？

Phase 2: Constraints (约束收集)
├── 平台信息：目标平台是什么？
├── 技术栈：使用什么技术栈？
├── 限制条件：有哪些限制条件？
└── 交付要求：交付格式、时间要求、质量标准？

Phase 3: Synthesis (生成执行)
├── 信息齐了之后才允许开始生成
└── 严格按照收集的信息执行
```

### 门控机制

**门控指令**：
- 在 Discovery 和 Constraints 阶段，不允许进入 Synthesis
- 必须显式确认信息已收集完整
- 使用门控检查清单验证信息完整性

**门控检查清单示例**：
```markdown
## 信息收集完成检查清单

### Discovery 完成？
- [ ] 问题核心已确认
- [ ] 目标输出已明确
- [ ] 任务背景已理解

### Constraints 完成？
- [ ] 平台信息已确认
- [ ] 技术栈已明确
- [ ] 限制条件已收集
- [ ] 交付要求已确认

### 允许开工？
- 只有所有项都勾选后才能进入 Synthesis 阶段
```

## 工作流程

1. **接收任务**：Agent 接收模糊的初始需求
2. **Discovery 阶段**：
   - Agent 提问："你到底要解决什么问题？"
   - 用户回答问题核心和目标
3. **Constraints 阶段**：
   - Agent 提问："目标平台是什么？使用什么技术栈？"
   - 用户回答约束条件
4. **门控检查**：检查信息是否收集完整
5. **Synthesis 阶段**：信息齐了之后开始生成

## 适用场景

- **模糊需求**：用户需求描述不清晰，缺少关键信息
- **缺少约束**：没有明确技术栈、平台、限制条件
- **复杂任务**：需要多种信息才能正确执行的任务
- **高风险任务**：信息不足可能导致严重后果的任务

## 优势

### 减少猜测错误

- 不依赖 Agent 的"自信猜测"
- 基于用户提供的确切信息执行

### 提高命中率

- 输出与真实需求对齐
- 减少反复修改的成本

### 用户参与感

- 用户明确知道 Agent 需要什么信息
- 用户参与决策过程，提高信任度

## 实现要点

### Discovery 阶段设计

**关键问题**：
- "你想要解决什么问题？"
- "期望的输出是什么样的？"
- "这个任务的背景是什么？"

### Constraints 阶段设计

**关键问题**：
- "目标平台是什么？（Web/Mobile/API）"
- "使用什么技术栈？（React/Vue/原生）"
- "有哪些限制条件？（性能要求/兼容性）"
- "交付要求是什么？（格式/时间/质量）"

### 门控检查设计

**检查机制**：
- 自动检查必填信息是否已收集
- 如果信息不全，阻止进入下一阶段
- 提示用户补充缺失信息

## 与其他模式组合

- **Inversion + Generator**：先问清需求，再用模板生成
- **Inversion + Pipeline**：先问清需求，再执行分步流程
- **Inversion + ToRapper**：问清技术栈后，加载对应知识

## 类比理解

### 类比项目管理

- **传统方式** = 项目经理凭经验估算需求，直接开工
- **Inversion 模式** = 项目经理先询问客户需求，确认后才开工

### 类比建筑设计

- **传统方式** = 设计师凭想象设计房屋
- **Inversion 模式** = 设计师先询问业主需求、预算、限制，再设计
