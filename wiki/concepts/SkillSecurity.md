## 定义

Skill 安全指在使用 Agent Skill 时需要注意的安全风险和防护措施。由于 Skill 可以运行脚本、访问文件系统、甚至连接外部网络，其强大能力也意味着潜在风险。Anthropic 提出三条安全建议：只安装可信来源的技能、使用前仔细阅读所有文件、理解执行逻辑。

## 安全风险来源

### Skill 的强大能力

- **运行脚本**：可以执行任意代码
- **访问文件系统**：可以读取、修改、删除文件
- **连接外部网络**：可以访问外部 API、数据库、网络资源

### 风险类型

- **恶意代码**：脚本包含恶意逻辑
- **数据泄露**：访问敏感文件并上传
- **权限滥用**：执行超出预期的操作
- **供应链攻击**：依赖的外部资源被篡改

## 三条安全建议

### 1. 只安装可信来源的技能

**来源审查**：
- 官方提供的 Skill
- 知名开源社区的 Skill
- 企业内部开发的 Skill
- 经过安全审计的 Skill

**避免来源**：
- 不明来源的第三方 Skill
- 没有代码审查的 Skill
- 缺少维护和更新的 Skill

### 2. 使用前仔细阅读所有文件

**重点审查**：
- **脚本依赖**：需要哪些外部库和工具
- **外部资源链接**：访问哪些外部服务和 API
- **文件访问路径**：读取和修改哪些文件
- **网络请求**：发送什么数据到哪个地址

**审查方法**：
- 阅读所有 `.py`、`.sh` 脚本文件
- 检查所有 `.json` 配置文件
- 确认外部链接的目标和用途
- 验证依赖的安全性和版本

### 3. 理解执行逻辑

**明确区分**：
- **哪些代码需要运行**：实际执行的脚本
- **哪些只是文档说明**：参考性的文档和注释

**执行前验证**：
- 在沙箱环境测试 Skill 行为
- 监控首次运行的实际操作
- 检查日志和输出是否符合预期

## 安全哲学

**核心理念**：安全不是限制能力，而是保护信任。

- **能力与风险并存**：Skill 的强大是双刃剑
- **信任需要验证**：不盲目信任任何 Skill
- **安全支持创新**：在安全边界内发挥 Skill 价值

## 企业级安全实践

### 技能仓库管理

- **内部仓库**：企业维护的可信 Skill 仓库
- **审核流程**：新 Skill 必须通过安全审查
- **版本控制**：跟踪 Skill 的变更历史

### 权限隔离

- **最小权限原则**：只授予 Skill 必需的权限
- **沙箱执行**：在隔离环境运行不可信 Skill
- **审计日志**：记录所有 Skill 执行操作

### 持续监控

- **行为分析**：监控 Skill 的实际行为
- **异常告警**：检测异常的文件访问或网络请求
- **定期审查**：定期审计已安装 Skill 的安全性

## 类比理解

### 类比浏览器扩展

- **Skill** = 浏览器扩展（可以访问网页内容、发送网络请求）
- **安全建议** = 安装前查看权限、只安装官方商店扩展

### 类比移动应用

- **Skill** = 手机应用（可以访问文件、相机、通讯录）
- **安全建议** = 只从官方应用商店下载、查看权限申请
