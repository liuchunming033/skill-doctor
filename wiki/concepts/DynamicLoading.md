## 定义

动态加载（Dynamic Loading）是 Agent Skill 的关键机制，指 Skill 不再固定在系统提示中，而是可被智能体根据任务需求动态检索、挂载和使用。通过动态加载，Agent 的智能边界不再受固定上下文窗口限制。

## 核心机制

### 工作流程

1. **任务识别**：Agent 接收任务，判断需要哪些能力
2. **Skill 发现**：Agent 从已装备技能清单中检索最相关的 Skill
3. **Skill 挂载**：Agent 通过 Tool 指挥虚拟机打开并读取 Skill 的 SOP 文件
4. **上下文注入**：Skill 内容返回 Agent 的上下文窗口
5. **学习决策**：Agent 学习 Skill 内容并决定下一步操作

### 与固定系统提示的区别

| 维度 | 固定系统提示 | 动态加载 Skill |
|------|-------------|---------------|
| 上下文占用 | 永久占用，无法释放 | 按需加载，任务结束后可释放 |
| 灵活性 | 固定不变，难以调整 | 动态发现，按任务选择 |
| 扩展性 | 受上下文窗口限制 | 理论上无限扩展 |
| 粒度 | 粗粒度，包含所有知识 | 细粒度，按场景精准加载 |

## 核心优势

### 突破上下文窗口限制

- **传统方式**：所有知识固定在系统提示中，受上下文窗口限制
- **动态加载**：按需加载相关 Skill，其他 Skill 不占用上下文
- **扩展性**：理论上可装备无限数量的 Skill，每次只加载相关的

### 精准化任务执行

- **场景匹配**：Agent 主动发现最适合当前任务的 Skill
- **专业能力**：加载特定 Skill 后，Agent 在该场景下表现更专业
- **输出质量**：输出更精准、更一致、更可复现

### 类比理解

类比操作系统加载程序：
- **固定系统提示** = 把所有程序都常驻内存（内存有限，程序数量受限）
- **动态加载 Skill** = 按需加载程序到内存（硬盘可存大量程序，运行时只加载需要的）
