## 定义

Skill 与 MCP（Model Context Protocol）的关系是互补而非重复。Skill 提供任务知识（How），教 Claude 如何完成一类工作，是思维的框架；MCP 提供外部连接（What），让 Claude 可以访问数据库、API 和软件资源，是行动的手臂。两者结合实现混合式智能：既懂得做事的方法，又能真正动手执行。

## 互补关系解析

| 维度 | Skill | MCP |
|------|-------|-----|
| 核心问题 | How（如何做） | What（能访问什么） |
| 提供内容 | 任务知识、SOP、方法论 | 外部连接、API、数据源 |
| 功能定位 | 思维的框架 | 行动的手臂 |
| 示例 | 教 Claude 如何审查合同 | 连接企业数据库和合同管理系统 |
| 能力类型 | 过程知识 + 组织背景 | 外部资源访问权限 |
| 组合效果 | 混合式智能：既懂方法又能执行 |

## 组合应用场景

### 场景 1：合同审查

**Skill 提供方法论**：
- 审查合同的 SOP
- 风险字段检查清单
- 合规性验证流程

**MCP 提供外部连接**：
- 连接企业数据库
- 访问合同管理系统
- 调用风险评估 API

**完整流程**：
1. Skill 指导如何审查（方法论）
2. MCP 连接数据库提取合同（资源访问）
3. Skill 指导比对风险字段（方法论）
4. MCP 调用风险评估 API（资源访问）
5. Skill 指导生成审查报告（方法论）

### 场景 2：数据分析报告

**Skill 提供方法论**：
- 数据分析的 SOP
- 指标计算方法
- 报告撰写框架

**MCP 提供外部连接**：
- 连接数据库读取原始数据
- 调用图表生成 API
- 访问报告发布系统

### 场景 3：客户服务

**Skill 提供方法论**：
- 客户问题分类逻辑
- 回复模板和话术
- 升级流程 SOP

**MCP 提供外部连接**：
- 连接 CRM 系统查询客户信息
- 访问知识库检索答案
- 调用工单系统创建工单

## 架构关系

### 分层架构

```
┌─────────────────────────────────────┐
│          Claude (决策层)            │
│  判断任务，选择 Skill，调用 MCP     │
└─────────────────────────────────────┘
         ↓                    ↓
    ┌─────────┐          ┌─────────┐
    │  Skill  │          │   MCP   │
    │ (方法论)│          │(资源层) │
    └─────────┘          └─────────┘
         ↓                    ↓
    SOP/脚本/模板        API/数据库/服务
```

### 协作机制

1. **任务识别**：Claude 识别任务类型
2. **Skill 选择**：Claude 选择相关 Skill 获取方法论
3. **MCP 调用**：按照 Skill 指导，调用 MCP 访问外部资源
4. **执行决策**：Claude 综合 Skill 方法和 MCP 资源做决策
5. **输出结果**：基于方法论和资源生成最终输出

## 核心价值

### 能力完整性

- **方法论**：Skill 提供如何做的知识
- **资源**：MCP 提供访问什么的权限
- **完整性**：方法 + 资源 = 可执行的完整能力

### 模块化设计

- **Skill 可替换**：同一 MCP 连接，不同的 Skill 提供不同方法论
- **MCP 可替换**：同一 Skill 方法论，不同的 MCP 连接不同系统
- **灵活组合**：按需组合 Skill 和 MCP

### 知识与权限分离

- **知识层**：Skill 封装业务知识，不需要关心具体系统
- **权限层**：MCP 管理外部访问权限，不包含业务逻辑
- **关注点分离**：降低耦合，提高可维护性

## 未来演进

### 更深度的集成

- Skill 直接引用 MCP 连接
- 自动发现可用的 MCP 资源
- 动态组合 Skill 和 MCP

### 标准化协议

- Skill 和 MCP 的标准化接口
- 跨平台的互操作性
- 社区共享的 Skill-MCP 组合

### 智能编排

- Claude 自动判断需要哪些 Skill 和 MCP
- 自动编排多个 Skill 和 MCP 的协作
- 自适应调整组合策略

## 类比理解

### 类比软件开发

- **Skill** = 设计模式和架构方法（如何设计系统）
- **MCP** = 数据库驱动和 API 客户端（如何连接外部系统）
- **组合** = 用设计模式指导开发，用驱动连接数据库

### 类比企业运营

- **Skill** = 业务流程和 SOP（如何开展业务）
- **MCP** = IT 系统和工具平台（访问什么系统）
- **组合** = 用 SOP 指导工作，用系统支撑执行
