## 定义

Agent + 人协作的软件交付流水线（AgentHumanPipeline）：在 AI Native 时代，将 AI Agent 作为流水线中有明确角色和边界的高能力节点嵌入研发流程，同时让人类回归需要判断力、创造力、决策力的环节，实现人机协同的新型研发模式。

## 关键信息

**为什么需要流水线（三重根本价值）：**

1. **定义生产秩序**：定义信息如何流转、谁在哪个节点负责什么、什么条件下可以进入下一阶段——这是 AI 无法自动建立的秩序。
2. **规模化复用**：将 Agent 能力封装为组织级资产，实现能力复用而非工具散用，解决 Prompt、模板、最佳实践散落在个人层面的问题。
3. **可度量可改进**：提供数据基础设施，让管理者能看到软件交付全貌，识别浪费和卡点，定向优化。

**Agent 进入流水线的三重价值：**

1. **组织效率提升**：Agent 承担重复性工作（格式化、基础 Bug 检测、测试生成、文档初稿），流水线吞吐按机器节拍运行。
2. **质量稳定提升**：AI 审查覆盖率远超人工，各环节通过自动质量门禁，重点环节人工审批。
3. **交付确定性增强**：过程可管理、结果可审计、投入产出可量化。

**企业实践模式（从文章案例提炼）：**

- [[Google]]：75% AI 代码 + 严格 CI/CD + AI 代码审查机制
- [[ApolloIO]]：双轨审查（AI BugBot + 人工）+ 自定义规则 + 92% PR 覆盖
- [[Accenture]]：小批量交付纪律 + 审查者赋能 + PR周期缩短75%

**核心论点**：AI Native 时代正确的解法不是"砍掉流水线"，而是让 Agent 进入流水线，成为流水线中有明确角色和边界的高能力节点。
